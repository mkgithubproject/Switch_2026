# 🔥 IGW vs NAT Gateway (Core Difference)

| Feature                  | **Internet Gateway (IGW)** | **NAT Gateway (NAT)**               |
| ------------------------ | -------------------------- | ----------------------------------- |
| Purpose                  | Connect VPC to Internet    | Allow **private subnet → internet** |
| Direction                | ✅ Inbound + Outbound       | ❌ Inbound, ✅ Outbound only          |
| Used by                  | Public Subnet              | Private Subnet                      |
| Public IP needed on EC2? | Yes                        | No                                  |
| Security                 | No filtering               | No filtering                        |
| IP Translation           | Yes (1:1 mapping)          | Yes (many → one public IP)          |

---

# 🧠 Simple Definition

👉 **IGW** = Direct internet access (2-way traffic)
👉 **NAT** = Safe internet access (only outgoing)

---

# 🔥 Example 1: IGW (Public EC2)

### Setup:

* EC2 in Public Subnet
* Public IP: `3.110.x.x`
* Route:

  ```
  0.0.0.0/0 → IGW
  ```

---

## 🚀 Flow

### 📥 Incoming (User → EC2)

```id="2u2k6j"
User → IGW → EC2
```

* IGW converts:

  ```
  3.110.x.x → 10.0.1.5
  ```

---

### 📤 Outgoing (EC2 → Internet)

```id="o15rxg"
EC2 → IGW → Internet
```

* IGW converts:

  ```
  10.0.1.5 → 3.110.x.x
  ```

---

## 🧠 Key Idea

👉 EC2 is **publicly reachable**

---

# 🔥 Example 2: NAT Gateway (Private EC2)

### Setup:

* EC2 in Private Subnet
* No public IP
* Route:

  ```
  0.0.0.0/0 → NAT Gateway
  ```

---

## 🚀 Flow

### 📤 Outgoing only

```id="gooqi1"
EC2 → NAT → IGW → Internet
```

* NAT converts:

  ```
  10.0.2.5 → 3.120.x.x (NAT’s IP)
  ```

---

### 📥 Incoming?

```id="k8aizs"
Internet ❌→ EC2
```

👉 Not allowed

---

## 🧠 Key Idea

👉 EC2 is **hidden (secure)**
👉 Can only **initiate requests**

# 🔥 Real-World Analogy (Never Forget)

### 🏠 IGW = House on main road

* Anyone can come
* You can go out
* Fully open

---

### 🏢 NAT = Office behind security gate

* Employees can go out
* Outsiders cannot enter directly

# 🔥 Combined Flow (Real Production)

```id="kqxvkm"
User → Load Balancer (Public Subnet, IGW)
     → EC2 App (Private Subnet)
     → NAT → Internet (if needed)
```

---

# ⚠️ Common Mistakes

❌ “NAT allows inbound traffic” → WRONG
❌ “Private EC2 can receive internet traffic via NAT” → WRONG
❌ “IGW provides security” → WRONG

---

# 🎯 Interview One-Liner

👉 “IGW enables direct bidirectional internet access for public resources, while NAT Gateway allows private subnet resources to access the internet securely for outbound traffic only, without exposing them to inbound connections.”
