# 🧠 What is an IAM Role?

👉 An **IAM Role** is an identity in AWS that:

* ❌ Has **no permanent user/password**
* ✅ Is **assumed temporarily**
* ✅ Has **permissions attached**

---

# 🔥 Real Example Scenario

👉 You (Mohit) are a developer
👉 You don’t have admin access directly
👉 But you can **assume a role → AdminRole**

---

# 📦 Full Role Structure (2 Parts)

An IAM Role has **2 policies**:

1. **Trust Policy** → Who can assume
2. **Permission Policy** → What they can do

---

# 📜 1. Trust Policy (WHO can assume)

```json id="trust-full"
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "AWS": "arn:aws:iam::123456789012:user/Mohit"
      },
      "Action": "sts:AssumeRole"
    }
  ]
}
```

---

## 🔍 Line-by-Line

### `"Version": "2012-10-17"`

👉 Policy language version (standard in AWS)

---

### `"Statement"`

👉 List of rules

---

### `"Effect": "Allow"`

👉 अनुमति दी गई है (Permission granted)

---

### `"Principal"`

```json id="trust-principal"
"AWS": "arn:aws:iam::123456789012:user/Mohit"
```

👉 WHO can assume?

✔️ Only user **Mohit**

---

### `"Action": "sts:AssumeRole"`

👉 WHAT can he do?

✔️ He can **assume (switch into) this role**

---

# 🧠 Meaning (Trust Policy)

👉 “Mohit is allowed to assume this role”

---

---

# 📜 2. Permission Policy (WHAT role can do)

```json id="perm-full"
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "ec2:TerminateInstances"
      ],
      "Resource": "*"
    }
  ]
}
```

---

## 🔍 Line-by-Line

### `"Effect": "Allow"`

👉 Permissions granted

---

### `"Action"`

```json id="perm-actions"
[
  "s3:GetObject",
  "ec2:TerminateInstances"
]
```

👉 Allowed actions:

✔️ Read S3 files
✔️ Terminate EC2 instances

---

### `"Resource": "*"`

👉 On which resources?

✔️ ALL resources (very open)

---

# 🧠 Meaning (Permission Policy)

👉 “Whoever assumes this role can:

* Read S3 files
* Terminate EC2 instances”

---

---

# 🔄 FULL FLOW (Step-by-Step)

## Step 1: Mohit has this policy

```json id="user-policy"
{
  "Effect": "Allow",
  "Action": "sts:AssumeRole",
  "Resource": "arn:aws:iam::123456789012:role/AdminRole"
}
```

---

## Step 2: Mohit runs command

```bash id="cmd-role"
aws sts assume-role --role-arn arn:aws:iam::123456789012:role/AdminRole
```

---

## Step 3: AWS checks

✔️ Trust Policy → allows Mohit
✔️ User Policy → allows assume role

---

## Step 4: AWS returns

🔑 Temporary credentials

---

## Step 5: Now Mohit becomes AdminRole

👉 He can:

✔️ Download S3 files
✔️ Terminate EC2
