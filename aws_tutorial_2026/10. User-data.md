Let’s do this properly with a **Node.js example using User Data**, step by step, so you can visualize exactly what happens.

# 🧠 Definition (clear and interview-ready)

👉 **User Data is a startup script that automatically runs on an EC2 instance (in Amazon EC2) during its first boot to install, configure, and start applications without manual intervention.**

---

# 🎯 Goal Example

👉 You launch EC2
👉 It automatically:

* Installs Node.js
* Creates a simple server
* Starts it

👉 When you hit browser:

```bash
http://<EC2-IP>:3000
```

👉 You see: **Hello from Node.js**

---

# 📜 Step 1: User Data Script

Paste this while launching EC2:

```bash id="node-user-data"
#!/bin/bash

# Update system
yum update -y

# Install Node.js
curl -sL https://rpm.nodesource.com/setup_18.x | bash -
yum install -y nodejs

# Create app directory
mkdir -p /home/ec2-user/app
cd /home/ec2-user/app

# Create Node.js server file
cat << 'EOF' > app.js
const http = require('http');

const server = http.createServer((req, res) => {
  res.write("Hello from Node.js EC2 🚀");
  res.end();
});

server.listen(3000, () => {
  console.log("Server running on port 3000");
});
EOF

# Start the server
node app.js &
```
