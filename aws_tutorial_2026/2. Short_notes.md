# AWS Basics Notes

## ☁️ Cloud Types

### 🔒 Private Cloud

* Used **within an organization**
* Organization is responsible for:

  * Infrastructure
  * Data servers
  * Maintenance
* No dependency on AWS or any public provider

---

### 🌍 Public Cloud

* Anyone can create and use resources
* Provider (like AWS) is responsible for:

  * Infrastructure setup
  * Data centers
  * Hardware maintenance

---

### 🔐 Private Cloud inside Public Cloud

* You can create an isolated environment inside AWS
* Example: **VPC (Virtual Private Cloud)**

---

## 👤 IAM (Identity & Access Management)

### 👥 Users (Authentication)

* Created for login purposes
* Represents a person or service
* Used to **authenticate** into AWS

---

### 📜 Policies (Authorization)

* Define **what actions are allowed**
* Written in **JSON format**

**Example:**

```json
{
  "Effect": "Allow",
  "Action": "s3:GetObject",
  "Resource": "arn:aws:s3:::my-bucket/*"
}
```

---

### 👨‍👩‍👧 Groups

* Collection of users
* Used for **efficiency**
* Instead of attaching policies one by one:

  * Create a group
  * Attach policies to group
  * Add users to group

---

### 🎭 Roles

* Similar to users but:

  * **Temporary access**
  * No username/password
* Common use cases:

  * Communication between AWS accounts
  * Access from private cloud to public cloud
  * Services accessing other services

---

## 🔑 Policies vs Permissions

### 📜 Policy

* A **set of rules** (JSON format)
* Example:

  * Allow `s3:GetObject` on `my-bucket/*`

---

### ✅ Permission

* What a user can actually do
* Example:

  * User can **read S3 files**

---

## 🌐 Route 53 (DNS Service)

* Maps **domain name → IP address**
* Helps DNS servers locate your server

---

## 🌍 Networking Concepts (VPC)

### 🏠 Real-World Analogy

| AWS Component | Real World            |
| ------------- | --------------------- |
| VPC           | Society               |
| EC2           | Your room             |
| Public IP     | Your house address    |
| IGW           | Society security gate |

---

### 🌐 Internet Gateway (IGW)

* Enables **two-way internet access**
* Used for **public resources**

---

### 🔄 NAT Gateway

* Enables **one-way internet access**
* Used for **private resources**
* Private instances can:

  * Access internet
  * But **internet cannot access them**

---

### 🧭 Route Table

* Controls **outbound traffic**
* Decides:

  * Where traffic should go
  * Via IGW or NAT

---

## 🧠 Quick Summary (Never Forget)

* **User = Login**
* **Policy = Rules**
* **Permission = What user can do**
* **Group = Collection of users**
* **Role = Temporary access**
