# What is a Load Balancer in AWS?

Imagine you opened a restaurant.

Suddenly 10,000 customers come.

If only **one waiter** handles everyone:

* customers wait
* system becomes slow
* waiter crashes 😵

So you hire multiple waiters.

Now one manager stands at the entrance and says:

* "You go to waiter 1"
* "You go to waiter 2"
* "You go to waiter 3"

That **manager = Load Balancer**

---

# Real AWS Meaning

A Load Balancer:

* receives client requests
* distributes traffic across multiple servers
* prevents one server from overloading
* increases availability and scalability

In AWS, servers are usually:

* EC2 instances
* containers
* ECS tasks
* Lambda functions
* Kubernetes pods

---

# Never Forget Analogy

# Mall Entry Gate

Think:

* Customers = Users
* Shops = Servers
* Main Security Gate = Load Balancer

Instead of everyone entering one shop directly:

```text
Users
   ↓
Load Balancer
 ↙   ↓   ↘
S1   S2   S3
```

Load balancer decides:

* where traffic goes
* which server is healthy
* which server is busy
* which server is dead

---

# Why Load Balancer Needed?

Without Load Balancer:

```text
Users → One Server
```

Problems:

* server crash
* downtime
* slow response
* no scalability

With Load Balancer:

```text
Users → LB → Many Servers
```

Benefits:

✅ High availability
✅ Fault tolerance
✅ Auto scaling support
✅ Better performance
✅ Zero downtime deployments
✅ SSL termination
✅ Routing control

---

# AWS Elastic Load Balancer (ELB)

AWS service name:

# Amazon Web Services Elastic Load Balancer (ELB)

ELB automatically distributes incoming traffic.

AWS manages:

* scaling
* availability
* failover
* health checks

You do NOT manually manage the LB server.

---

# Types of AWS Load Balancers

AWS mainly provides:

| Type                            | Layer     | Used For            |
| ------------------------------- | --------- | ------------------- |
| Application Load Balancer (ALB) | Layer 7   | HTTP/HTTPS          |
| Network Load Balancer (NLB)     | Layer 4   | TCP/UDP             |
| Gateway Load Balancer (GWLB)    | Layer 3/4 | Security appliances |
| Classic Load Balancer (CLB)     | Old       | Legacy              |

---

# First Understand OSI Layers

Very important.

# Layer 7 → Application Layer

Understands:

* URLs
* HTTP headers
* cookies
* domains

Example:

```text
/api/users
/images
/admin
```

---

# Layer 4 → Transport Layer

Understands only:

* IP
* Port
* TCP/UDP

Does NOT understand:

* URL
* API path
* headers

# ALB Understands:

Suppose you have:

```text

```

ALB can route traffic like:

```text

/payments   → Payment Service
```

# Health Checks in ALB

ALB continuously checks:

```text
GET /health
```

If server unhealthy:

❌ stop sending traffic

---

# Example

Server 2 crashed:

```text
ALB
 ↙     ↘
S1      S2 ❌
```

ALB automatically sends all traffic to S1.

---

# Target Group

Very important concept.

# What is Target Group?

A logical group of backend servers.

Example:

```text
Target Group:
Order-Service-TG

Contains:
EC2-1
EC2-2
EC2-3
```

ALB sends traffic to target group.

---

# Load Balancing Algorithms

AWS mainly uses:

## Round Robin

```text
Req1 → S1
Req2 → S2
Req3 → S3
Req4 → S1
```

---

# Least Outstanding Requests

Traffic goes to least busy server.

---

# Sticky Sessions

Normally:

```text
Req1 → S1
Req2 → S2
```

But some apps need same user always on same server.

ALB uses cookies for this.

---

# SSL Termination

Without LB:

Every server handles HTTPS encryption.

With ALB:

```text
User HTTPS
    ↓
ALB decrypts SSL
    ↓
HTTP internally
```

Benefits:

* less CPU load on EC2
* easier certificate management

---

# Auto Scaling + ALB

This is VERY important in interviews.

---

# Without Auto Scaling

Traffic spike:

```text
100 users → server crash
```

---

# With Auto Scaling + ALB

```text
Traffic increases
       ↓
Auto Scaling creates EC2s
       ↓
ALB automatically sends traffic
```

---

# Full Architecture

```text
Users
   ↓
Route53
   ↓
ALB
   ↓
Auto Scaling Group
   ↓
EC2 Instances
```

Components:

* Route53 = DNS
* ALB = traffic manager
* ASG = create/remove servers
* EC2 = actual app servers

# 2. Network Load Balancer (NLB)

Works at:

# Layer 4

Understands:

* TCP
* UDP
* TLS

Does NOT understand:

* URL
* path
* headers

---

# NLB is Used When:

You need:

✅ ultra high performance
✅ millions of requests/sec
✅ very low latency
✅ gaming
✅ VoIP
✅ real-time systems

---

# Example

Gaming server.

Packets must move FAST.

NLB works almost at network level.

---

# Difference Between ALB and NLB

| Feature         | ALB      | NLB                         |
| --------------- | -------- | --------------------------- |
| Layer           | 7        | 4                           |
| Understands URL | Yes      | No                          |
| HTTP Routing    | Yes      | No                          |
| TCP/UDP         | No       | Yes                         |
| Speed           | Fast     | Ultra Fast                  |
| Best For        | Web Apps | High performance networking |

---

# Example Comparison

## ALB

```text
/api/users → Server A

```

Because it understands HTTP.

---

## NLB

Only sees:

```text
IP + Port
```

Nothing else.

# Static IP in NLB

NLB supports:

✅ static IP

# Internal vs Internet Facing Load Balancer

---

# Internet Facing LB

Publicly accessible.

```text
Internet → LB → Servers
```

Has public IP/DNS.

Used for:

* websites
* APIs

---

# Internal LB

Private only.

```text
Frontend → Internal LB → Backend services
```

No public access.

Used inside VPC.

---

# Cross Zone Load Balancing

Suppose:

```text
AZ1 → 2 servers
AZ2 → 8 servers
```

Without cross-zone:

Traffic uneven.

With cross-zone:

LB distributes evenly across ALL servers.

---

# Availability Zone Example

```text
          Load Balancer
         ↙             ↘
     AZ1                 AZ2
   EC2-1               EC2-3
   EC2-2               EC2-4
```

If AZ1 fails:

ALB routes to AZ2.

This gives:

# High Availability

# Important Interview Question

# Difference Between Reverse Proxy and Load Balancer?

Load balancer can act as reverse proxy.

Reverse proxy mainly:

* hides backend servers
* forwards requests

Load balancer additionally:

* distributes traffic
* health checks
* scaling
