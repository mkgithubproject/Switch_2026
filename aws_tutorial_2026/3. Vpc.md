# 🧠 What is `10.0.0.0/16`?

👉 It’s a **CIDR block** used to define a network range in a VPC.

It has 2 parts:

* **10.0.0.0** → Starting IP (network address)
* **/16** → Network size

---

# 🧩 What does `/16` mean?

Use this formula:

\text{Host bits} = 32 - 16 = 16 \quad\Rightarrow\quad \text{Total IPs} = 2^{16} = 65536

👉 So:

* Total IPs = **65,536**

---

# 🏙️ Never Forget Analogy

👉 `10.0.0.0/16` = **Big city**

* City name → `10.0`
* Houses → remaining part (`x.x`)

👉 Range:

```
10.0.0.0 → 10.0.255.255
```

---

# 🔢 How IP Range Expands

In `/16`:

* First **2 parts fixed** → `10.0`
* Last **2 parts change**

👉 So:

```
10.0.0.0
10.0.0.1
10.0.0.2
...
10.0.1.0
...
10.0.255.255
```

# 🧠 Step 1: IP in Binary

Take:

```text
10.0.0.0
```

Convert each octet (8 bits each):

```text
10  = 00001010
0   = 00000000
0   = 00000000
0   = 00000000
```

👉 Full 32-bit IP:

```text
00001010.00000000.00000000.00000000
```

---

# 🧩 Step 2: What `/16` Means

👉 `/16` = first **16 bits are fixed**, rest **16 bits are free**

```text
[ Network (fixed) ] [ Host (free) ]

00001010.00000000 | 00000000.00000000
   (10)     (0)        (X)      (X)
```

---

# 🔥 Step 3: Find the LAST IP

To get the **maximum range**, set all free bits = 1

```text
00001010.00000000 | 11111111.11111111
```

---

# 🧠 Step 4: Convert back to Decimal

Now convert `11111111`:

11111111_2 = 2^8 - 1 = 255

👉 So:

```text
11111111 = 255
```

---

# 🎯 Final Result

```text
Start IP → 10.0.0.0
End IP   → 10.0.255.255
```

---

# 🏙️ Never Forget Visual

```text
10.0. [00000000 → 11111111] . [00000000 → 11111111]
        (0 → 255)             (0 → 255)
```

---

# 💥 Golden Line (Never Forget)

👉 **"All 1s in binary = 255 in decimal."**

# ⚡ Super Simple Rule

* Free bits = host bits
* Max value = all 1s
* All 1s (8 bits) = 255

# 🧠 When you create EC2 directly, what AWS gives by default?

AWS uses a **default VPC** in your account (from **Amazon VPC**).

---

# ✅ What is created by default

When you launch an EC2 **without custom setup**, AWS uses:

### ✔️ 1. Default VPC

* Already exists in your account

---

### ✔️ 2. Default Subnet

* One subnet per Availability Zone
* EC2 is launched inside one of them

---

### ✔️ 3. Internet Gateway (IGW)

* Already attached to default VPC

---

### ✔️ 4. Route Table

* Already configured:

  ```
  0.0.0.0/0 → IGW
  ```

👉 That’s why your EC2 gets internet access

---

### ✔️ 5. Security Group

* Default SG:

  * Allows all outbound
  * Inbound mostly blocked (unless you allow)

---

# ❌ What is NOT created by default

### ❌ NAT Gateway

* NOT created automatically
* You must create it manually

👉 Because:

* NAT costs money 💸
