## 1. 🏙️ VPC = Your Society

---

## 2. 🛣️ Subnets = Roads inside Society

You divide society into zones:

### 🌐 Public Subnet

* Connected to internet
* Example:

  * Web server here

### 🔒 Private Subnet

* No direct internet access
* Example:

  * Database here

---

## 3. 🚪 IGW (Internet Gateway) = Main Gate

* Allows **two-way traffic**
* Internet ↔ Your VPC

👉 If attached:

* People can come IN
* People can go OUT

---

## 4. 🔄 NAT Gateway = Secret Exit Door

* Only **outgoing traffic**
* Internet ❌ cannot come inside

👉 Use case:

* Private server needs:

  * Updates
  * API calls

---

## 5. 🧭 Route Table = Traffic Police

Decides **where traffic goes**

### Example Rules:

#### Public Subnet:

```
0.0.0.0/0 → IGW
```

👉 Means:

* Send all internet traffic via IGW

---

#### Private Subnet:

```
0.0.0.0/0 → NAT Gateway
```

👉 Means:

* Only outbound internet allowed

---

## 6. 🛡️ Security Group = Personal Security Guard

* Works at **instance level**
* Stateful (remembers request)

### Example:

* Allow:

  * Port 80 (HTTP)
  * Port 22 (SSH)

👉 Only allowed traffic enters

---

## 7. 🚧 NACL (Network ACL) = Society Rules

* Works at **subnet level**
* Stateless (checks every request)

### Example:

* Allow:

  * Port 80
* Deny:

  * Specific IP

# 🧠 What is `10.0.0.0/16`?

👉 It’s a **CIDR block** used to define a network range in a VPC.

It has 2 parts:

* **10.0.0.0** → Starting IP (network address)
* **/16** → Network size

---

# 🧩 What does `/16` mean?

Use this formula:

\text{Host bits} = 32 - 16 = 16 \quad\Rightarrow\quad \text{Total IPs} = 2^{16} = 65536

👉 So:

* Total IPs = **65,536**

---

# 🏙️ Never Forget Analogy

👉 `10.0.0.0/16` = **Big city**

* City name → `10.0`
* Houses → remaining part (`x.x`)

👉 Range:

```
10.0.0.0 → 10.0.255.255
```

---

# 🔢 How IP Range Expands

In `/16`:

* First **2 parts fixed** → `10.0`
* Last **2 parts change**

👉 So:

```
10.0.0.0
10.0.0.1
10.0.0.2
...
10.0.1.0
...
10.0.255.255
```

# 🧠 Step 1: IP in Binary

Take:

```text
10.0.0.0
```

Convert each octet (8 bits each):

```text
10  = 00001010
0   = 00000000
0   = 00000000
0   = 00000000
```

👉 Full 32-bit IP:

```text
00001010.00000000.00000000.00000000
```

---

# 🧩 Step 2: What `/16` Means

👉 `/16` = first **16 bits are fixed**, rest **16 bits are free**

```text
[ Network (fixed) ] [ Host (free) ]

00001010.00000000 | 00000000.00000000
   (10)     (0)        (X)      (X)
```

---

# 🔥 Step 3: Find the LAST IP

To get the **maximum range**, set all free bits = 1

```text
00001010.00000000 | 11111111.11111111
```

---

# 🧠 Step 4: Convert back to Decimal

Now convert `11111111`:

11111111_2 = 2^8 - 1 = 255

👉 So:

```text
11111111 = 255
```

---

# 🎯 Final Result

```text
Start IP → 10.0.0.0
End IP   → 10.0.255.255
```

---

# 🏙️ Never Forget Visual

```text
10.0. [00000000 → 11111111] . [00000000 → 11111111]
        (0 → 255)             (0 → 255)
```

# 💥 Golden Line (Never Forget)

👉 **"All 1s in binary = 255 in decimal."**

# ⚡ Super Simple Rule

* Free bits = host bits
* Max value = all 1s
* All 1s (8 bits) = 255

👉 **“/16 → last 16 bits free → max = 255.255”**
