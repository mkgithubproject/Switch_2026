# 🧠 VPC — Never Forget Concept

## 🏙️ Imagine This

You build your **own private society inside AWS**.

* You control:

  * Who enters 🚪
  * Who goes out 🌍
  * Which room talks to internet 📡

👉 That society = **VPC (Virtual Private Cloud)**

| Real World            | AWS            |
| --------------------- | -------------- |
| Society               | VPC            |

# 🔥 Step-by-Step Story (Never Forget Flow)

## 1. 🏙️ VPC = Your Society

* Private isolated network
* Example:

  * CIDR: `10.0.0.0/16` (like society area)

---

## 2. 🛣️ Subnets = Roads inside Society

You divide society into zones:

### 🌐 Public Subnet

* Connected to internet
* Example:

  * Web server here

### 🔒 Private Subnet

* No direct internet access
* Example:

  * Database here

---

## 3. 🚪 IGW (Internet Gateway) = Main Gate

* Allows **two-way traffic**
* Internet ↔ Your VPC

👉 If attached:

* People can come IN
* People can go OUT

---

## 4. 🔄 NAT Gateway = Secret Exit Door

* Only **outgoing traffic**
* Internet ❌ cannot come inside

👉 Use case:

* Private server needs:

  * Updates
  * API calls

---

## 5. 🧭 Route Table = Traffic Police

Decides **where traffic goes**

### Example Rules:

#### Public Subnet:

```
0.0.0.0/0 → IGW
```

👉 Means:

* Send all internet traffic via IGW

---

#### Private Subnet:

```
0.0.0.0/0 → NAT Gateway
```

👉 Means:

* Only outbound internet allowed

---

## 6. 🛡️ Security Group = Personal Security Guard

* Works at **instance level**
* Stateful (remembers request)

### Example:

* Allow:

  * Port 80 (HTTP)
  * Port 22 (SSH)

👉 Only allowed traffic enters

---

## 7. 🚧 NACL (Network ACL) = Society Rules

* Works at **subnet level**
* Stateless (checks every request)

### Example:

* Allow:

  * Port 80
* Deny:

  * Specific IP

# 🚀 Real Practical Example

## 🎯 Goal:

Host a secure web app

---

## 🏗️ Setup:

### Step 1: Create VPC

```
10.0.0.0/16
```

---

### Step 2: Create Subnets

* Public Subnet:

  ```
  10.0.1.0/24
  ```
* Private Subnet:

  ```
  10.0.2.0/24
  ```

---

### Step 3: Attach IGW

* Connect VPC to internet

---

### Step 4: Route Tables

#### Public Route Table:

```
0.0.0.0/0 → IGW
```

#### Private Route Table:

```
0.0.0.0/0 → NAT
```

---

### Step 5: Launch EC2

* Public Subnet:

  * Web server 🌐

* Private Subnet:

  * Database 🔒

---

### Step 6: Security

* Web Server:

  * Allow HTTP (80)
  * Allow SSH (22)

* DB Server:

  * Allow only from web server

---

# 🧠 Final Memory Trick (Golden Rule)

## 🔥 Remember This Line:

👉 **"Public subnet talks directly to internet, private subnet talks through NAT."**

---

## 🧩 One-Line Summary of Each

* **VPC** → Your private network
* **Subnet** → Division of network
* **IGW** → Internet entry/exit
* **NAT** → One-way internet
* **Route Table** → Traffic direction
* **Security Group** → Instance firewall
* **NACL** → Subnet firewall
