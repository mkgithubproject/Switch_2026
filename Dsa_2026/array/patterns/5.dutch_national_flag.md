# Main Logic

At every step:

Check:

```text id="dnf15"
arr[mid]
```

---

# CASE 1 → arr[mid] == 0

0 belongs LEFT side.

Swap:

```text id="dnf16"
arr[low] ↔ arr[mid]
```

Then:

```text id="dnf17"
low++
mid++
```

---

# CASE 2 → arr[mid] == 1

1 belongs middle.

Do nothing.

Just:

```text id="dnf18"
mid++
```

---

# CASE 3 → arr[mid] == 2

2 belongs RIGHT side.

Swap:

```text id="dnf19"
arr[mid] ↔ arr[high]
```

Then:

```text id="dnf20"
high--
```

IMPORTANT:

```text id="dnf21"
Do NOT increment mid here
```

Because swapped element still unprocessed.

VERY IMPORTANT.

---

# FULL CODE (MOST IMPORTANT)

```javascript id="dnf22"
function sortColors(arr) {

    let low = 0;
    let mid = 0;
    let high = arr.length - 1;

    while (mid <= high) {

        // CASE 1
        if (arr[mid] === 0) {

            [arr[low], arr[mid]] = [arr[mid], arr[low]];

            low++;
            mid++;
        }

        // CASE 2
        else if (arr[mid] === 1) {

            mid++;
        }

        // CASE 3
        else {

            [arr[mid], arr[high]] = [arr[high], arr[mid]];

            high--;
        }
    }

    return arr;
}

console.log(sortColors([2,0,2,1,1,0]));
```

Output:

```text id="dnf23"
[0,0,1,1,2,2]
```

# Your Question

You asked:

```text id="q1"
If arr[mid] == 0,
why do we do mid++ ?

What if low has 2 ?
```

Very smart observation.

---

# Short Answer

Because:

```text id="q2"
Everything before low is already guaranteed to be 0.
```

So:

```text id="q3"
arr[low] can NEVER be 2
```

at that moment.

That is the KEY invariant.

---

# MOST IMPORTANT CONCEPT

Dutch National Flag works because we MAINTAIN RULES.

These rules are called:

# Invariants

---

# Invariant Meaning

At every moment:

```text id="q4"
0 to low-1       → all 0s
low to mid-1     → all 1s
mid to high      → unknown
high+1 to end    → all 2s
```

Visualization:

```text id="q5"
| 0s | 1s | unknown | 2s |
        ^
       mid
```

---

# SUPER IMPORTANT

When:

```text id="q6"
arr[mid] == 0
```

We swap:

```text id="q7"
arr[low] ↔ arr[mid]
```

Now your doubt is:

```text id="q8"
What if arr[low] was 2?
```

Impossible.

Why?

Because:

```text id="q9"
low is NEVER inside 2s region
```

---

# Let's Prove It

Suppose:

```text id="q10"
low = 2
mid = 4
```

Then region is:

```text id="q11"
0..1 → all 0s
2..3 → all 1s
4..high → unknown
```

Meaning:

```text id="q12"
arr[low]
```

belongs to:

```text id="q13"
1s region
```

OR:

Sometimes low == mid.

But NEVER 2.

---

# IMPORTANT CASES

# Case 1 → low == mid

Example:

```text id="q14"
low = 0
mid = 0
```

Swap with itself.

No issue.

Then:

```text id="q15"
mid++
low++
```

Safe.

---

# Case 2 → low < mid

Then:

```text id="q16"
arr[low]
```

must be:

```text id="q17"
1
```

because:

```text id="q18"
low..mid-1 region = all 1s
```

So after swapping:

```text id="q19"
1 comes to mid
```

And that's already processed correctly.

So:

```text id="q20"
mid++
```

is safe.

---

# THIS IS THE ENTIRE MAGIC

When swapping with low:

```text id="q21"
we KNOW incoming value at mid is safe
```

But when swapping with high:

```text id="q22"
incoming value is UNKNOWN
```

That’s why:

# For 0

```text id="q23"
mid++
```

# For 2

```text id="q24"
DON'T do mid++
```
