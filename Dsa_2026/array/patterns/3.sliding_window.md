# 3. Sliding Window

This is one of the MOST IMPORTANT interview patterns.

If you master this pattern, you can solve MANY array and string problems quickly.

Asked heavily in Amazon, Google, Adobe, Walmart interviews.

---

# What is Sliding Window?

Sliding Window means:

> Maintain a moving range/window over array or string.

Instead of recalculating everything again and again.

---

# Real Life Analogy (Never Forget)

Imagine train window.

You only see some people at a time.

Window moves:

```text id="s0ps8t"
[1 2 3] 4 5 6
1 [2 3 4] 5 6
1 2 [3 4 5] 6
```

Window keeps sliding.

That is:

```text id="rk9h0c"
SLIDING WINDOW
```

---

# Why Sliding Window Exists

Without sliding window:

You repeatedly calculate same things.

Very expensive.

Sliding window reuses previous work.

# Most Common Use Case

# Maximum Sum Subarray of Size K

---

# Problem

Given:

```text id="2jlwmh"
arr = [2,1,5,1,3,2]
k = 3
```

Find maximum sum of any subarray of size 3.

---

# All Windows

```text id="qpf7jm"
[2,1,5] = 8
[1,5,1] = 7
[5,1,3] = 9
[1,3,2] = 6
```

Answer:

```text id="y3f6mv"
9
```

---

# Brute Force Approach

Nested loops.

```text id="n4c0h7"
O(n²)
```

---

# Sliding Window Optimization

Key idea:

Instead of recalculating entire sum:

```text id="zeblko"
Subtract outgoing element
Add incoming element
```

---

# Visual Understanding

Initial window:

```text id="w3e9qh"
[2,1,5] = 8
```

Move window right:

Remove:

```text id="6f6x0i"
2
```

Add:

```text id="x4s34h"
1
```

New sum:

```text id="v9z4ya"
8 - 2 + 1 = 7
```

Again:

Remove:

```text id="pp1a5d"
1
```

Add:

```text id="egm8d0"
3
```

New sum:

```text id="zjlwm3"
7 - 1 + 3 = 9
```

# Code (VERY IMPORTANT)

```javascript id="5ylm7m"
function maxSumSubarray(arr, k) {

    let windowSum = 0;

    // first window
    for (let i = 0; i < k; i++) {
        windowSum += arr[i];
    }

    let maxSum = windowSum;

    // slide window
    for (let i = k; i < arr.length; i++) {

        windowSum = windowSum - arr[i - k] + arr[i];

        maxSum = Math.max(maxSum, windowSum);
    }

    return maxSum;
}

console.log(maxSumSubarray([2,1,5,1,3,2], 3));
```

Output:

```text id="j4h6od"
9
```

# Sliding Window Formula

\text{New Window Sum} = \text{Old Window Sum} - \text{Outgoing Element} + \text{Incoming Element}

---

# Fixed Size Sliding Window

Window size stays constant.

Example:

```text id="vfp98h"
size = 3 always
```

Problems:

* max sum of size k
* average of size k
* first negative in window

---

# Variable Size Sliding Window

Window grows/shrinks dynamically.

VERY IMPORTANT in interviews.

Problems:

* longest substring
* smallest subarray
* unique characters
* at most k distinct

---

# Variable Window Idea

Expand right pointer.

If duplicate found:

```text id="vb2jcx"
shrink from left
```

This creates dynamic sliding window.

# Time Complexity

Most sliding window problems:

```text id="u7x8lx"
O(n)
```

Because each element enters/exits window once.

# Common Mistakes

---

## Mistake 1

Wrong outgoing index.

Wrong:

```javascript id="6ryjlwm"
arr[i]
```

Correct:

```javascript id="1smy5m"
arr[i - k]
```

---

## Mistake 2

Forgetting first window initialization.

---

## Mistake 3

Confusing:

```text id="l4rkqt"
fixed window
```

with

```text id="94b8x3"
variable window
```
