# Refined Problem Statement

You are given:

* an integer array `nums`
* an integer `target`

Your task is to find **two different elements** in the array whose sum equals `target`.

Return the indices of those two elements.

---

# Important Conditions

* Each input has **exactly one valid solution**
* You cannot use the same element twice
* The returned indices can be in any order

---

# Approaches for Two Sum

Problem:
Find two different indices such that:

```js id="f4m8qa"
nums[i] + nums[j] = target
```

Return those indices.

---

# Approach 1 — Brute Force

## Idea

Check every possible pair.

For each element:

* compare with all remaining elements
* if sum equals target → return indices

---

# Code

```js id="k7v2pd"
function twoSum(nums, target) {
    for (let i = 0; i < nums.length; i++) {

        for (let j = i + 1; j < nums.length; j++) {

            if (nums[i] + nums[j] === target) {
                return [i, j];
            }

        }
    }
}
```

# Approach 2 — Hash Map (Best Approach)

## Idea

Instead of searching every pair:

* store visited numbers in a hash map
* calculate needed value:

```js id="n5v1qb"
complement = target - currentNumber
```

* if complement already exists → answer found

---

# Why This Works

Example:

```js id="c3m8ty"
target = 9
current = 7
```

Needed:

```js id="j6q2wp"
9 - 7 = 2
```

If `2` already exists in map:
→ pair found

---

# Code

```js id="a8v4mk"
function twoSum(nums, target) {

    let map = {};

    for (let i = 0; i < nums.length; i++) {

        let complement = target - nums[i];

        if (map.hasOwnProperty(complement)) {
            return [map[complement], i];
        }

        map[nums[i]] = i;
    }
}
```

# Approach 3 — Using Map Object

Instead of plain object, use JavaScript `Map`.

---

# Code

```js id="w8p3tc"
function twoSum(nums, target) {

    let map = new Map();

    for (let i = 0; i < nums.length; i++) {

        let complement = target - nums[i];

        if (map.has(complement)) {
            return [map.get(complement), i];
        }

        map.set(nums[i], i);
    }
}
```

---

# Why `Map` is Better Than Object

| Object             | Map               |
| ------------------ | ----------------- |
| String keys mainly | Any datatype keys |
| Older approach     | Modern approach   |
| Prototype issues   | Cleaner API       |

If the array is **sorted**, we can use a much better approach:

# Two Pointer Approach

Instead of using:

* nested loops
* hash maps

we use:

```js id="t8q4vn"
left pointer + right pointer
```

---

# Why It Works

Because array is sorted.

Example:

```js id="q3m7kp"
nums = [1,2,3,4,6]
target = 6
```

We can intelligently move pointers.

---

# Idea

* Start:

  * `left` at beginning
  * `right` at end

* Calculate:

```js id="h6v2ra"
sum = nums[left] + nums[right]
```

---

# Rules

### If sum == target

✅ answer found

---

### If sum < target

Need bigger sum:

```js id="f1x8tm"
left++
```

because sorted array increases toward right.

---

### If sum > target

Need smaller sum:

```js id="y4q7wc"
right--
```

because sorted array decreases toward left.

---

# Code

```js id="n9m3pk"
function twoSum(nums, target) {

    let left = 0;
    let right = nums.length - 1;

    while (left < right) {

        let sum = nums[left] + nums[right];

        if (sum === target) {
            return [left, right];
        }

        else if (sum < target) {
            left++;
        }

        else {
            right--;
        }
    }
}
```
