Yes.

During development:

```bash
npm run dev
```

Vite converts JSX → JavaScript **on demand (runtime of development server)**.

Not during browser runtime.

Not during build.

Let's understand with a real project.

---

# File Structure

```text
my-react-app/
│
├── index.html
│
├── src/
│   ├── main.jsx
│   ├── App.jsx
│   └── components/
│       └── User.jsx
│
├── package.json
└── vite.config.js
```

---

# User Component

```jsx
// src/components/User.jsx

function User() {
  return <h1>Hello Mohit</h1>;
}

export default User;
```

---

# App Component

```jsx
// src/App.jsx

import User from "./components/User";

function App() {
  return (
    <div>
      <User />
    </div>
  );
}

export default App;
```

---

# main.jsx

```jsx
// src/main.jsx

import ReactDOM from "react-dom/client";
import App from "./App";

ReactDOM
  .createRoot(document.getElementById("root"))
  .render(<App />);
```

---

# Browser Opens

You visit:

```text
http://localhost:5173
```

---

# Request 1

Browser asks:

```http
GET /
```

Vite returns:

```html
<!-- index.html -->

<body>
  <div id="root"></div>

  <script
    type="module"
    src="/src/main.jsx">
  </script>
</body>
```

---

# Request 2

Browser sees:

```html
<script src="/src/main.jsx">
```

and asks:

```http
GET /src/main.jsx
```

---

# Important Part

Browser DOES NOT understand:

```jsx
render(<App />)
```

because that's JSX.

So before sending the file:

```text
Vite intercepts request
        ↓
Uses esbuild
        ↓
Converts JSX → JS
        ↓
Sends transformed file
```

---

# What Browser Receives

Your code:

```jsx
render(<App />);
```

Vite sends something similar to:

```js
render(
  React.createElement(App)
);
```

(or React's newer JSX runtime)

Browser never sees raw JSX.

---

# Then Browser Executes

```js
ReactDOM
  .createRoot(root)
  .render(
     React.createElement(App)
  );
```

---

# App Executes

React calls:

```js
App();
```

returns:

```jsx
<div>
  <User />
</div>
```

---

# User Executes

React calls:

```js
User();
```

returns:

```jsx
<h1>Hello Mohit</h1>
```

---

# React Element Tree

React creates objects like:

```js
{
  type: App
}
```

↓

```js
{
  type: User
}
```

↓

```js
{
  type: "h1",
  children: "Hello Mohit"
}
```

---

# Fiber Creation

React converts those elements into fibers.

```text
Root Fiber
    |
 App Fiber
    |
 User Fiber
    |
  h1 Fiber
```

Stored in memory.

No DOM yet.

---

# DOM Creation

React creates:

```js
const h1 =
 document.createElement("h1");

h1.textContent =
 "Hello Mohit";
```

---

# Commit Phase

React inserts:

```js
root.appendChild(h1);
```

Now:

```html
<div id="root">
  <h1>Hello Mohit</h1>
</div>
```

---

# Browser Paint

Screen:

```text
Hello Mohit
```

# Full Development Flow

```text
You Write JSX
      ↓

npm run dev
      ↓

Vite Server Starts
      ↓

Browser GET /
      ↓

index.html Returned
      ↓

Browser GET main.jsx
      ↓

Vite Converts JSX → JS
      ↓

Browser Executes JS
      ↓

React Executes Components
      ↓

React Elements Created
      ↓

Fiber Tree Created
      ↓

DOM Nodes Created
      ↓

Commit Phase
      ↓

Browser Paint
      ↓

UI Visible
```

# What About Build?

When you run:

```bash
npm run build
```

Vite does JSX conversion **once**.

Before build:

```jsx
render(<App />);
```

After build:

```js
render(jsxs(...));
```

inside:

```text
dist/assets/index-abc123.js
```

Now browser downloads already-transformed JavaScript.

So:

```text
Development
-----------
Browser Request
      ↓
Vite converts JSX every time file requested

Production
-----------
Build converts JSX once
      ↓
Browser downloads ready JS
```

### Never Forget

Think of Vite as a translator.

**Development:**

```text
Browser: "Give me main.jsx"

Vite:
"Wait, browser can't read JSX"

JSX → JS

Send JS
```

**Production:**

```text
npm run build

Vite:
"Let me translate everything now"

Creates dist/

Browser:
Downloads already translated JS
```

That's why Vite feels fast: it doesn't bundle the whole app during development; it transforms only the files the browser requests.
