# React Lifecycle (Interview + Real Understanding)

When someone says **"React Lifecycle"**, the most accepted answer is:

```text
1. Mounting
2. Updating
3. Unmounting
```

These describe the **life of a component**.

---

# Visualize a Component as a Human

```text
Birth      → Mounting
Life       → Updating
Death      → Unmounting
```

---

# 1. Mounting (Birth)

Component appears for the first time.

Example:

```jsx
function App() {
  return <User />;
}

function User() {
  return <h1>Hello Mohit</h1>;
}
```

Before:

```text
Nothing on screen
```

After:

```text
Hello Mohit
```

Lifecycle:

```text
User Mounted
```

---

## What React Does Internally

```text
Execute Component
      ↓
Create Fiber
      ↓
Create DOM Nodes
      ↓
Attach DOM to Real DOM
      ↓
Mounted
```

---

## Hook

```jsx
useEffect(() => {
  console.log("Mounted");
}, []);
```

Runs once.

---

# 2. Updating (Life Changes)

Component already exists.

Data changes.

Example:

```jsx
function Counter() {

  const [count, setCount] =
    useState(0);

  return (
    <button
      onClick={() =>
        setCount(count + 1)
      }
    >
      {count}
    </button>
  );
}
```

---

Initial:

```text
0
```

Click:

```text
1
```

Click:

```text
2
```

Click:

```text
3
```

Component still exists.

Only data changed.

Lifecycle:

```text
Counter Updated
```

---

## What React Does Internally

```text
setCount()
      ↓
Trigger
      ↓
Render
      ↓
Commit
      ↓
Updated
```

---

## Hook

```jsx
useEffect(() => {
  console.log("Updated");
}, [count]);
```

Runs whenever count changes.

---

# 3. Unmounting (Death)

Component removed from screen.

Example:

```jsx
{
  show && <User />
}
```

Initially:

```text
User Visible
```

Then:

```js
setShow(false);
```

Now:

```text
User Removed
```

Lifecycle:

```text
User Unmounted
```

---

## What React Does Internally

```text
Render
      ↓
Mark Fiber For Deletion
      ↓
Cleanup Effects
      ↓
Remove DOM
      ↓
Unmounted
```

---

## Hook

```jsx
useEffect(() => {

  return () => {
    console.log("Unmount");
  };

}, []);
```

Runs before component is removed.

# Modern React Lifecycle vs Internal Phases

Lifecycle (what interviewer usually means):

```text
Mounting
Updating
Unmounting
```

Internal Fiber Processing:

```text
Trigger
Render
Commit
```

Relation:

```text
Mounting
   └─ Trigger → Render → Commit

Updating
   └─ Trigger → Render → Commit

Unmounting
   └─ Trigger → Render → Commit → Cleanup
```

---

# Interview Answer (Perfect)

> React components go through three lifecycle stages:
>
> 1. **Mounting** – when the component is created and added to the DOM.
> 2. **Updating** – when state, props, or context changes cause a re-render.
> 3. **Unmounting** – when the component is removed from the DOM and cleanup occurs.
>
> In modern React, these lifecycle stages are implemented using Hooks (`useEffect`) and internally processed by the React Fiber architecture through Trigger, Render, and Commit phases.

### One-line memory trick

```text
Mount = Component appears
Update = Component changes
Unmount = Component disappears
```

If you remember only that line, you'll never get confused in interviews.
