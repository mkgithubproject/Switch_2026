Perfect. Let's go deeper than most React tutorials.

We'll take this component:

```jsx
function User() {
  return <h1>Hello Mohit</h1>;
}
```

And:

```jsx
ReactDOM.createRoot(
  document.getElementById("root")
).render(<User />);
```

---

# Initial Situation

Browser already has:

```html
<body>
  <div id="root"></div>
</body>
```

DOM Tree:

```text
body
 └── div#root
```

React has not mounted anything yet.

---

# Step 1: Execute Component

React starts:

```js
User();
```

Actually React doesn't care that it's JSX.

It simply calls your function.

```js
function User() {
  return <h1>Hello Mohit</h1>;
}
```

becomes roughly:

```js
function User() {
  return React.createElement(
    "h1",
    null,
    "Hello Mohit"
  );
}
```

returns:

```js
{
  type: "h1",
  props: {
    children: "Hello Mohit"
  }
}
```

This is a React Element.

Think:

```text
Blueprint
NOT DOM
```

Memory:

```text
React Element

{
 type: "h1",
 children: "Hello Mohit"
}
```

No DOM created yet.

---

# Step 2: Create Fiber

This is where React gets interesting.

React converts the React Element into a Fiber.

Conceptually:

```js
{
  tag: "HostComponent",
  type: "h1",
  pendingProps: {
    children: "Hello Mohit"
  }
}
```

Fiber stores:

```text
What component is this?
Who is parent?
Who is child?
Any state?
Any effects?
Priority?
```

Think:

```text
React Element
      ↓
Fiber
```

Example Fiber:

```text
User Fiber
      |
      v
h1 Fiber
```

Memory:

```text
Fiber Tree

User Fiber
    |
    └── h1 Fiber
```

Still:

```text
No DOM
```

---

# Why Fiber Exists?

Without Fiber:

```text
React cannot:
  Pause
  Resume
  Prioritize
  Schedule
```

Fiber is React's work unit.

Think:

```text
Fiber = React's Task Object
```

---

# Step 3: Create DOM Nodes

Now React reaches Commit Phase.

Fiber:

```text
h1 Fiber
```

becomes:

```js
document.createElement("h1");
```

Internally something like:

```js
const h1 =
  document.createElement("h1");

h1.textContent =
  "Hello Mohit";
```

Memory:

```text
DOM Node Created

<h1>Hello Mohit</h1>
```

Still not visible.

Why?

Because it isn't attached.

Think:

```text
Created
Not Inserted
```

---

# Step 4: Attach to Real DOM

React now performs:

```js
root.appendChild(h1);
```

Conceptually.

DOM becomes:

```html
<div id="root">
  <h1>Hello Mohit</h1>
</div>
```

Now browser sees:

```text
DOM changed
```

---

# Step 5: Browser Paint

Browser rendering pipeline:

```text
DOM
 ↓
Layout
 ↓
Paint
 ↓
Composite
```

Pixels appear:

```text
Hello Mohit
```

on screen.

---

# Step 6: Component Mounted

Now React considers:

```text
Component Mounted
```

because:

```text
Fiber exists
DOM exists
DOM attached
UI visible
```

Now React runs:

```jsx
useEffect(() => {
  console.log("Mounted");
}, []);
```

# Real Hidden Structure (Simplified)

After mount, React memory looks roughly like:

```text
Fiber Tree

Root Fiber
    |
User Fiber
    |
h1 Fiber
```

Each Fiber contains things like:

```js
{
  type: "h1",
  stateNode: DOM_NODE,
  child: null,
  sibling: null,
  return: UserFiber,
  alternate: null,
  flags: 0
}
```

Where:

```js
stateNode
```

points to:

```html
<h1>Hello Mohit</h1>
```

actual DOM node.

Blueprint

Build wall

Workers build house parts.

```text
Walls created

```

The most important realization:

```text
React Element
     ≠ DOM

Fiber
     ≠ DOM

DOM Node
     ≠ Mounted

Mounted means:
Fiber created
+
DOM created
+
DOM attached to actual document
```

This is the exact sequence React follows during the initial mount.
