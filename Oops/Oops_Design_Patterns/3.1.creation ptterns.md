# 1️⃣ Singleton Pattern

---

## 🧠 Mental Model

> 🏦 “Only ONE instance allowed”

---

## 📌 When to Use

* Shared resource
* DB connection

---

## 💻 Practical Example (Node.js)

```js
class DB {
  constructor() {
    if (DB.instance) return DB.instance;

    this.connection = "Connected to MongoDB";
    DB.instance = this;
  }
}

const db1 = new DB();
const db2 = new DB();

console.log(db1 === db2); // true ✅
```

---

## 💡 Real Use

* MongoDB / Redis connection
* Logger

# 2️⃣ Factory Pattern

---

## 🧠 Mental Model

> 🏭 “Give input → get object”

---

## 📌 When to Use

* Many object types
* Avoid messy `if-else`

---

## 💻 Practical Example

```js
class Email {
  send() { console.log("Email sent"); }
}

class SMS {
  send() { console.log("SMS sent"); }
}

class NotificationFactory {
  static create(type) {
    if (type === "email") return new Email();
    if (type === "sms") return new SMS();
  }
}

const notification = NotificationFactory.create("email");
notification.send();
```

---

## 💡 Real Use

* Payment system
* Notification system

# 3️⃣ Abstract Factory Pattern

---

## 🧠 Mental Model

> 🏭🏭 “Factory of related objects”

---

## 📌 When to Use

* Create families of related objects

---

## 💻 Practical Example

```js
class MacUI {
  button() {
    return { render: () => console.log("Mac Button") };
  }
}

class WindowsUI {
  button() {
    return { render: () => console.log("Windows Button") };
  }
}

// Client code
function renderUI(factory) {
  const btn = factory.button();
  btn.render();
}

renderUI(new MacUI());
renderUI(new WindowsUI());
```

---

## 💡 Real Use

* Multi-platform apps
* Multi-database systems

# 4️⃣ Builder Pattern

---

## 🧠 Mental Model

> 🧱 “Build step-by-step like LEGO”

---

## 📌 When to Use

* Many optional fields
* Complex objects

---

## 💻 Practical Example

```js
class User {
  constructor(builder) {
    this.name = builder.name;
    this.age = builder.age;
    this.email = builder.email;
  }
}

class UserBuilder {
  setName(name) {
    this.name = name;
    return this;
  }

  setAge(age) {
    this.age = age;
    return this;
  }

  setEmail(email) {
    this.email = email;
    return this;
  }

  build() {
    return new User(this);
  }
}

const user = new UserBuilder()
  .setName("Mohit")
  .setAge(25)
  .build();

console.log(user);
```

---

## 💡 Real Use

* API payload creation
* Query builders

# 5️⃣ Prototype Pattern

---

## 🧠 Mental Model

> 🧬 “Clone instead of create”

---

## 📌 When to Use

* Object creation is heavy
* Need duplicates

---

## 💻 Practical Example

```js
const baseUser = {
  role: "user",
  greet() {
    console.log("Hello");
  }
};

const user1 = Object.create(baseUser);
user1.name = "Mohit";

const user2 = Object.create(baseUser);
user2.name = "Rahul";

user1.greet();
user2.greet();
```

---

## 💡 Real Use

* Object cloning
* Performance optimization
