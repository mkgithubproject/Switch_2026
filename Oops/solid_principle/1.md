# 🚀 What is SOLID?

**SOLID = 5 principles to write clean, scalable, maintainable code**

| Letter | Meaning                         |
| ------ | ------------------------------- |
| S      | Single Responsibility Principle |
| O      | Open/Closed Principle           |
| L      | Liskov Substitution Principle   |
| I      | Interface Segregation Principle |
| D      | Dependency Inversion Principle  |

---

# 🧠 First, Understand This (VERY IMPORTANT)

👉 Bad code = Hard to change
👉 Good code (SOLID) = Easy to extend without breaking

Think like this:

> “If tomorrow requirement changes… will my code break or adapt?”

---

# 1️⃣ S — Single Responsibility Principle (SRP)

## 📌 Rule:

👉 A class/function should have **ONLY ONE reason to change**

---

## ❌ Bad Example (Realistic Node.js)

```js
class UserService {
  createUser(user) {
    // save to DB
  }

  sendEmail(user) {
    // send email
  }

  generateReport(user) {
    // generate PDF
  }
}
```

🚨 Problem:

* If email logic changes → modify same class
* If report changes → modify same class
* Everything is mixed 😵

---

## ✅ Good Example

```js
class UserService {
  createUser(user) {
    // save to DB
  }
}

class EmailService {
  sendEmail(user) {
    // send email
  }
}

class ReportService {
  generateReport(user) {
    // generate PDF
  }
}
```

---

## 🧠 Real Life Example

👉 One person:

* Chef 👨‍🍳
* Driver 🚗
* Doctor 🩺

❌ Wrong
👉 One person = One role
✅ Correct

---

## 💡 Memory Trick

👉 “**1 class = 1 job**”

---

# 2️⃣ O — Open/Closed Principle (OCP)

## 📌 Rule:

👉 Code should be **open for extension** but **closed for modification**

---

## ❌ Bad Example

```js
function getDiscount(userType) {
  if (userType === "regular") return 10;
  if (userType === "premium") return 20;
}
```

🚨 Problem:
👉 Every new type → modify function 😩

---

## ✅ Good Example

```js
class Discount {
  getDiscount() {}
}

class Regular extends Discount {
  getDiscount() {
    return 10;
  }
}

class Premium extends Discount {
  getDiscount() {
    return 20;
  }
}
```

---

## 🧠 Real Life Example

👉 Mobile Apps:

* You **add new features**
* But don’t rewrite whole app

---

## 💡 Memory Trick

👉 “**Add new code, don’t change old code**”

---

# 3️⃣ L — Liskov Substitution Principle (LSP)

## 📌 Rule:

👉 Child class should **replace parent without breaking code**

---

## ❌ Bad Example

```js
class Bird {
  fly() {}
}

class Penguin extends Bird {
  fly() {
    throw new Error("Cannot fly");
  }
}
```

🚨 Problem:
👉 Penguin is not behaving like Bird

---

## ✅ Good Example

```js
class Bird {}

class FlyingBird extends Bird {
  fly() {}
}

class Penguin extends Bird {
  swim() {}
}
```

---

## 🧠 Real Life Example

👉 If I say:
“Bring a bird that flies”

❌ Penguin breaks expectation
✅ Eagle works

---

## 💡 Memory Trick

👉 “**Child should behave like parent**”

---

# 4️⃣ I — Interface Segregation Principle (ISP)

## 📌 Rule:

👉 Don’t force classes to implement unused methods

---

## ❌ Bad Example

```js
class Machine {
  print() {}
  scan() {}
  fax() {}
}

class Printer extends Machine {
  fax() {
    throw Error("Not supported");
  }
}
```

---

## ✅ Good Example

```js
class Printer {
  print() {}
}

class Scanner {
  scan() {}
}
```

---

## 🧠 Real Life Example

👉 TV Remote:

* You don’t want 100 useless buttons

---

## 💡 Memory Trick

👉 “**Keep things small & specific**”

---

# 5️⃣ D — Dependency Inversion Principle (DIP)

## 📌 Rule:

👉 High-level modules should NOT depend on low-level modules
👉 Both should depend on abstraction

---

## ❌ Bad Example

```js
class MySQL {
  connect() {}
}

class UserService {
  constructor() {
    this.db = new MySQL();
  }
}
```

🚨 Problem:
👉 Tight coupling → cannot switch DB

---

## ✅ Good Example

```js
class Database {
  connect() {}
}

class MySQL extends Database {
  connect() {}
}

class MongoDB extends Database {
  connect() {}
}

class UserService {
  constructor(db) {
    this.db = db;
  }
}
```

---

## 🧠 Real Life Example

👉 Switch:

* Works with any bulb (Philips, Havells)

---

## 💡 Memory Trick

👉 “**Depend on interface, not concrete class**”
